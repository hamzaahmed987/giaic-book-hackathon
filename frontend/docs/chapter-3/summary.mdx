# Chapter 3 Summary

## Key Techniques

| Technique | When to Use |
|-----------|-------------|
| **Few-Shot** | Pattern recognition tasks |
| **Chain of Thought** | Complex reasoning |
| **Role Playing** | Domain expertise needed |
| **Output Formatting** | Structured data extraction |

## CRAFT Framework Quick Reference

- **C**ontext: Set the scene
- **R**ole: Define the persona
- **A**ction: Specify the task
- **F**ormat: Define output structure
- **T**one: Set communication style

## Common Mistakes

1. Being too vague
2. Not providing examples
3. Ignoring output format
4. Missing edge cases
